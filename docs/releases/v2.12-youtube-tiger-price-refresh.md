# v2.12 - YouTube Coverage, Tiger Recovery, and Price Reliability

**Release Date:** April 13, 2026

Version 2.12 expands the public chat surface with per-game YouTube coverage, documents the current Tiger recovery workflows more explicitly, and aligns pricing/operator docs with the latest storefront-authority fixes.

## Highlights

### YouTube Coverage In Chat

- `/chat` now supports explicit per-game YouTube coverage for tracked Steam games.
- The current views include latest matched uploads, creator coverage, top videos, fastest recent growth, content mix, and upload cadence.
- The chat UI can page these structured tables inline through the dedicated `/api/chat/youtube-coverage` route backed by the `getYoutubeGameCoverage` Tiger contract.

### Tiger Workflow Recovery

- Tiger preview and production sync docs now describe the live retry and fallback behavior rather than the earlier straight-through replay model.
- The current workflows capture the app-change watermark in the data-plane context, retry bounded `app_change_events` drift, and fall back to projection-only `exact_parity` when historical projection drift is the only remaining failure.
- Preview operators now have a faster events/news verification path with `tiger-preview-events-news.yml` and `stop_after_classification=true`.

### Storefront Authority And Pricing

- Effective app pricing now prefers storefront-backed `current_price_cents` and only shows sale state when there is a real non-free price to display.
- Operator docs now include the `repair-storefront-authority` workflow for repairing missing storefront-authoritative latest-state fields.
- PICS-side retry and cooldown behavior is now documented more clearly as enrichment/fallback logic, not as the source of truth for storefront pricing or `is_free`.

## What This Means For Operators

- Treat YouTube coverage as part of the current chat contract surface, not as a future-facing experiment doc.
- Use the updated Tiger deployment and GitHub Actions docs for current recovery behavior, especially during preview validation and projection drift incidents.
- Keep storefront authority rules in mind when debugging pricing, release, or free-to-play fields on `/apps`.
