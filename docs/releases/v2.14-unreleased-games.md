# v2.14 - Unreleased Games Tracker

**Release Date:** May 7, 2026

Version 2.14 adds the `/unreleased` workspace for tracking upcoming Steam games and identifying publisher opportunity before release.

## Highlights

- New `/unreleased` page for unreleased, upcoming, undated, and stale-date Steam games.
- Tiger-backed projection `metrics.unreleased_games_projection` optimized for fast filtering, sorting, stats, media, news, and change-intelligence signals.
- Opportunity Score for ranking likely outreach or publishing opportunities.
- Detail drawer with overview, media, timeline, and news tabs.
- Watch button powered by existing PublisherIQ pins.
- Column customization with URL-persisted visible column order.
- CSV export that follows the visible column order.
- Adult-content include/exclude/only filter.

## Data Plane

The feature depends on:

- `packages/data-plane/sql/tiger-bootstrap/0084_unreleased_games_page_projection.sql`
- `metrics.unreleased_games_projection`
- `metrics.unreleased_filter_counts`

The projection reads Tiger-side catalog, relationships, taxonomy, media, Steam news, and change-intelligence tables. It must be applied manually during an approved Tiger maintenance window and refreshed after relevant Tiger sync batches.

## UX Updates

- Row actions are placed beside the selection checkbox on the far left.
- `No Publisher` opportunity stats display self-published context where publisher and developer are the same.
- Steam news links use the stable store-news URL when a GID is available.
- Drawer media uses contained frames so odd screenshot and trailer dimensions remain visible.
- Timeline shows both change events and Steam news in one chronological stream.
- `Opp Score` fits in compact table headers and includes a methodology tooltip.

## Validation Evidence

Validated on the feature branch with:

```bash
pnpm --filter @publisheriq/admin test -- src/app/'(main)'/unreleased/lib/unreleased-columns.test.ts src/app/'(main)'/unreleased/lib/unreleased-export.test.ts
pnpm --filter @publisheriq/admin lint
pnpm --filter @publisheriq/admin build
```

The admin route smoke test confirmed `/unreleased` redirects unauthenticated users to login instead of returning a 404 or server error when the app is built.
